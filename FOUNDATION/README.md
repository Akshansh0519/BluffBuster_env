# FOUNDATION — Project specifications and workflows

This folder holds all planning and architecture documents for **The Examiner (BluffBuster)**. The repository root keeps only the hackathon-facing [`README.md`](../README.md).

## Contents

| File | Role |
|------|------|
| `guardrails.md` | Supreme authority — MSRs, scope, session rituals |
| `architecture.md` | Technical source of truth — reward spec, diagrams |
| `implementation_plan.md` | Phases, tasks, gates, paste-ready prompts |
| `implementation_coder1.md` | Environment & simulator (C1) playbook |
| `implementation_coder2.md` | Training, eval, deployment (C2) playbook |
| `implementation_validator.md` | Validator QC playbook |
| `merge_procedure.md` | Git branch and merge protocol |
| `project_structure.md` | Annotated file tree (code layout) |
| `submission_checklist.md` | Pre-submit MSR and judging checklist |
| `context_primer.md` | Short primer to paste into AI tools each session |
| `Base propmt.txt` | Original master prompt (source) |
